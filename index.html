The Omni Project

First, let me tell you that Omni is not about creating something new. Omni does not invent new programming language features. It does not introduce new syntax or concepts.

Instead, Omni changes the way programming languages work.

All programming languages consist of multiple steps to create an executable program or library. Every programming language's first step in creating a program or library is reading and understanding the source code that is stored in text files. Omni breaks with that concept. Code is no longer written in simple text files by (more or less) simple text editors. Instead, the code persists in a kind of database that stores the language constructs just as they are inserted by the programmer.

Advantages

  • No file clutter
  • No need for file naming conventions
  • No thoughts or arguments about which code belongs into which file
  • A whole project or set of projects is stored in a single file.
  • Inspect and edit the piece of code you are interested in, isolated from clutter around that code in the same file.
  • Jump/show declarations immediately, without the IDE to need to parse all code files to find the piece of code you want to jump to. Especially C++ programmers working on large projects know how bad this actually works, e.g. in Microsoft Visual Studio.
  • etc.
